Day 0: Planning & Contract Freeze
Duration: 2–4 Hours
No feature development begins before this phase is completed.

Step 4: Freeze API Contracts (P2 + P3)
P2 and P3 jointly define all backend APIs.
Examples:
POST /api/query
POST /api/agents/asset
POST /api/agents/diagnose
POST /api/agents/comply
They finalize:
Request JSON
Response JSON
Error responses
Streaming (SSE) event names
These interfaces remain stable throughout development.

Step 5: Freeze Shared Function Contracts (P2 + P3)
Shared functions are defined before implementation.
Example:
context_graph_query(
tag,
query,
depth
)
The input and output format is frozen.
Internal implementation can change later without affecting other modules.

Step 6: Create Development Fixture (P1)
P1 creates a small but realistic dataset containing:
10 fake equipment
30–50 knowledge facts
Neo4j nodes and relationships
Sample embeddings in Qdrant
Sample citations
This fixture allows every developer to start immediately without waiting for the complete ingestion
pipeline.

Step 7: Docker Environment (P4)
P4 prepares Docker Compose for the complete development environment.
Services include:
PostgreSQL
Neo4j
Qdrant
Redis
Backend
Frontend
Everyone now works in an identical environment.

End of Day 0
At this point:
Database schema is frozen
Graph ontology is frozen
API contracts are frozen
Function signatures are frozen
Docker environment is running
Development fixture is available
Now all four developers begin working independently.

Key Principle
The team does not depend on unfinished code from other developers.
Instead, everyone depends only on frozen contracts:
Database schema
Graph ontology
API specifications
Shared function signatures
Development fixtures
As long as these interfaces remain unchanged, each developer can build, test, and iterate
independently, enabling true parallel development throughout Stage 1.
